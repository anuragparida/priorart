# Phase 1 — Ship by Sunday Night (Weekend 1)

> **Tier:** Must-be. **Goal:** A working idea-lookup at `localhost:15173` (frontend) / `localhost:18000` (API), MRR ≥ 0.5 on a 100-idea labeled benchmark, README with a real leaderboard screenshot. No Temporal, no Dagster, no MLflow, no Langfuse — those are Phase 2. The whole point of Phase 1 is to *prove the retrieval and the comparison loop work* before adding the MLOps scaffolding.

This is the **minimum** that makes the CV line defensible. If Phase 1 isn't shippable, the project is a wrapper script.

---

## What ships at the end of Phase 1

- [ ] `uv`-managed FastAPI service running at `localhost:18000`.
- [ ] Postgres 16 + pgvector in Docker, with the YC public directory loaded and embedded.
- [ ] `POST /search` — ANN search against the corpus. Returns ranked IDs + similarity scores.
- [ ] `POST /ideas/analyze` — embed → ANN search → structured LLM compare (top-3) → market-scope stub → Pydantic-validated `IdeaVerdict` response.
- [ ] `evals/labeled_v100.jsonl` — 100 hand-labeled `{idea, expected_top_ids, is_duplicate}` triples. **Written first, before the system.**

- [ ] Vite + React + shadcn/ui dark-mode UI at `localhost:15173`. One page: idea input → ranked competitors with structured verdicts + market-scope badge + evidence links.

## Detailed task breakdown

Each task is sized to ~2–4 hours of focused work. Total Phase 1 budget: ~16 hours, doable in a weekend with breaks.

### 1.1 — Project bootstrap (1h)
- [ ] `uv init` in the project root. Pin Python 3.12.
- [ ] Add `pyproject.toml` dependencies: `fastapi`, `uvicorn[standard]`, `sqlalchemy>=2`, `pydantic>=2`, `psycopg[binary]`, `pgvector`, `sentence-transformers`, `instructor`, `anthropic`, `rank_bm25`, `duckdb`, `typer`, `pytest`.
- [ ] `mkdir -p src/api src/data src/llm src/eval src/frontend data/snapshots evals results configs`
- [ ] `docker-compose.yml` with: `postgres:16` (with `pgvector` image — use `pgvector/pgvector:pg16`), API service (placeholder for now), volumes for `data/`, `pgdata`.

### 1.2 — YC public directory scraper (2h)
- [ ] Scrape `https://www.ycombinator.com/companies` (and the per-company detail pages) for: name, 1-paragraph description, tags, batch, status (active/closed/acquired/inactive), URL.
- [ ] Output: `data/snapshots/yc_<date>.jsonl`, one record per line. UTF-8, deduped by `(name, batch)`.
- [ ] Manifest: `data/snapshots/yc_<date>.manifest.json` with `count`, `scrape_date`, `source_url`, `schema_version`.
- [ ] Idempotent: re-running on the same date produces a byte-identical file. Use deterministic sort order.
- [ ] **Verify before committing:** `wc -l data/snapshots/yc_*.jsonl` should be ~4–6K. If it's 0, the scraper broke.

### 1.3 — Postgres + pgvector schema + ingestion (2h)
- [ ] SQLAlchemy 2.x models: `Company { id, name, description, batch, status, url, tags[], source, snapshot_date }` and `CompanyEmbedding { company_id (FK), embedding vector(1024), model_version, created_at }`.
- [ ] HNSW index on `CompanyEmbedding.embedding` with `m=16, ef_construction=64`. Verify the index is created (`\d company_embeddings` in psql).
- [ ] Ingestion script: `python -m src.data.ingest --snapshot data/snapshots/yc_<date>.jsonl` — chunks descriptions by sentence, embeds with `bge-m3`, writes to `company_embeddings`. Idempotent on `(company_id, model_version)`.
- [ ] Health check: `psql -c "SELECT count(*) FROM company_embeddings"` should match the snapshot count.

### 1.4 — ANN search endpoint (1h)
- [ ] `POST /search {"query": "AI-powered legal contract review for SMB law firms", "top_k": 20}` → ranked list of `{id, name, description, similarity}`.
- [ ] Embed the query with the same `bge-m3` model. Cosine similarity via pgvector.
- [ ] Return both cosine score and a normalized 0–1 confidence for downstream thresholding.

### 1.5 — Labeled eval set v1 (3h, **do this early in the weekend, not at the end**)
- [ ] `evals/labeled_v100.jsonl` with 100 triples. Balanced:
  - 40 known-duplicates (pick 20 active YC companies, write 2 paraphrasings of each as a "new idea").
  - 30 known-novel (long-tail ideas with no close YC match — e.g. "an AI tool for composing Persian poetry," "a subscription service for vintage typewriters").
  - 30 adversarial (slight pivots, market overlap, similar-tech-different-domain).
- [ ] Each record: `{"id", "idea", "expected_top_ids": [list of company_ids], "is_duplicate": bool, "category": "duplicate" | "novel" | "adversarial_pivot" | ...}`.
- [ ] **Hand-labeled only.** No LLM labeling. Spend the time. This is the most important artifact in Phase 1.
- [ ] `evals/labeled_v100.README.md` explaining the label policy, the categories, the rationale per category.

### 1.6 — Eval harness v1 (2h)
- [ ] `python -m eval.run --benchmark evals/labeled_v100.jsonl --config configs/dense_bge_m3.yaml --output results/leaderboard.csv`.
- [ ] Computes: MRR, nDCG@10, precision@5, recall@10, FPR-on-novel (fraction of `is_duplicate=false` records where the system returned a top-1 with cosine > threshold).
- [ ] Writes CSV with one row per (config, metric) and a Markdown summary table.
- [ ] Threshold sweep baked in: runs at cosine thresholds [0.5, 0.55, 0.6, 0.65, 0.7, 0.75, 0.8], reports per-threshold metrics. Pick the threshold that maximizes MRR subject to FPR-on-novel ≤ 0.15.
- [ ] **Verify on the 100-idea benchmark.** MRR should be ≥ 0.5 with `bge-m3` dense on the YC corpus. If it's < 0.5, something is wrong with the ingestion or the embedding model — debug before moving on.

### 1.7 — Structured LLM comparison (2h)
- [ ] Pydantic schemas:
  ```python
  class CompetitorVerdict(BaseModel):
      company_id: int
      name: str
      similarity_axes: list[str]   # ["AI-assisted drafting", "SMB market", "subscription pricing"]
      key_differences: list[str]   # ["We focus on NDA review only", "Larger entity-extraction model"]
      likely_failure_modes: list[str]
      evidence_links: list[str]    # [company url, crunchbase url, etc.]
      confidence: float            # 0-1, model self-reported

  class MarketScope(str, Enum):
      WIDE_OPEN = "wide_open"
      CROWDED_BUT_GROWING = "crowded_but_growing"
      SATURATED = "saturated"
      NICHE_BUT_REAL = "niche_but_real"

  class IdeaVerdict(BaseModel):
      idea: str
      top_competitors: list[CompetitorVerdict]   # top-3
      market_scope: MarketScope
      market_scope_rationale: str
      supporting_evidence: list[str]
  ```
- [ ] Use `instructor` (or `outlines`) with Claude Sonnet 4.5 for constrained generation. Wrap in a `src.llm.compare.compare_topk(idea, top_k: list[Company]) -> list[CompetitorVerdict]` function.
- [ ] Cost-control: hard cap at top-3 (configurable). One LLM call per request, not per competitor.
- [ ] **Verify the JSON parses.** A bad schema-violation should produce a clear error, not a silently-truncated response.

### 1.8 — `/ideas/analyze` endpoint (1h)
- [ ] `POST /ideas/analyze {"idea": "..."}` orchestrates: embed → ANN search → `compare_topk` (top-3) → market-scope classification (single LLM call with the top-3 as context) → assemble `IdeaVerdict`.
- [ ] Returns the `IdeaVerdict` directly (Pydantic-validated). No 500s on schema-violation — surface as `{"error": "schema_violation", "details": ...}`.
- [ ] Health check: `GET /healthz` returns `{"status": "ok", "db": "ok", "model": "bge-m3", "corpus_count": N}`.

### 1.9 — Frontend (3h)
- [ ] `pnpm create vite@latest src/frontend -- --template react-ts`.
- [ ] Install Tailwind, shadcn/ui (`pnpm dlx shadcn@latest init`). Dark mode default.
- [ ] One page: `<IdeaAnalyzer />` — textarea, submit button, results panel.
- [ ] Results panel: ranked competitors (name + 1-line description), structured verdict (similarity axes as chips, key differences as bullets), market-scope badge with color coding, evidence links as anchor tags.
- [ ] Honest empty state: "No similar launches found in the YC corpus. This is genuinely novel — or outside the YC market."
- [ ] Loading state during LLM call (it'll be 5–15 seconds).
- [ ] Use your standard shadcn tokens; no custom design system.

### 1.10 — README + landscape + leaderboard screenshot (2h)
- [ ] README sections: 1-paragraph pitch, leaderboard screenshot (real, captured from your run), quickstart, methodology link, landscape table (from `docs/LANDSCAPE.md`), "how to add a retrieval config" guide.
- [ ] Capture the leaderboard: run the harness, screenshot the terminal output, save as `docs/assets/leaderboard-v1.png`.
- [ ] `docs/LANDSCAPE.md` — the expanded competitive-landscape table.
- [ ] `docs/METHODOLOGY.md` — what each metric means, how the benchmark was constructed, the label policy.
- [ ] GitHub topics on first push: `ai`, `rag`, `retrieval`, `evaluation`, `mlops`, `pgvector`, `startups`, `competitor-research`, `pydantic`.

### 1.11 — Smoke test + commit (30min)
- [ ] `make eval` runs end-to-end and produces a fresh `results/leaderboard.csv`.
- [ ] `make smoke` hits `/healthz` + `/search` + `/ideas/analyze` with a known input, asserts a non-empty response.
- [ ] Commit + push to `main`. No `main`-force, no squashing into a single commit.

## Pitfalls (Phase 1 specific)

- **Do not label the eval set with an LLM.** Spend the 3 hours hand-labeling. The whole benchmark is suspect if labels are LLM-generated.
- **Do not skip the threshold sweep.** A single threshold is a number, not a tuned system. The sweep is what makes the threshold defensible.
- **Do not use Cohere rerank as the default config in Phase 1.** No API calls for the default; keep the system offline-runnable. Add rerank in Phase 2 as a 4th opt-in config.
- **Do not add Temporal "to save time later."** Phase 1 is *just* the FastAPI + the LLM call. Temporal is Phase 2 work.
- **Do not optimize the frontend.** It needs to be functional and clean. Phase 3 is polish. If you're spending 30+ minutes on a UI animation, you're doing it wrong.
- **Do not write the LLM prompt in `constants.ts`.** Put it in `src/llm/prompts/compare.py` as a module constant. The prompt is code, not config.
- **Do not call the LLM 20 times per request.** Hard cap at top-3 (one LLM call). Cost and latency matter.
